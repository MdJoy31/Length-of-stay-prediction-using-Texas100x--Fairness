     → Only code 1 shown → "100% Male" FALSE

     → Both shown → True 36.6% / 63.4% split

Key Finding #2: RACE Fairness Breakthrough (+30pp DI)
──────────────────────────────────────────────────────

BEFORE AFCE:
   Black patients: ~45% predicted positive rate
   White patients: ~75% predicted positive rate
   → Disparate Impact DI = min/max = 0.45/0.75 = 0.60 ✗ UNFAIR

AFTER AFCE:
   Black patients: ~68% predicted (higher threshold 0.54 offsets lower base rate)
   White patients: ~66% predicted (lower threshold 0.47)
   → Disparate Impact DI = 0.66/0.82 = 0.80 ✓ FAIR (80% rule)

   How it works:
   - Black patients have lower historical 3-day+ baseline
   - Their default threshold 0.5 was too harsh
   - AFCE raises it to 0.54 (needs higher probability to accept)
   - Now both groups have similar prediction rates despite different baselines

Key Finding #3: Age Group Fairness (Limited but Honest)
───────────────────────────────────────────────────────

AGE_GROUP has fundamental fairness challenge:
   Young Adult (18-44): ~25% extended stays in reality
   Elderly (65+): ~60% extended stays in reality
   → 2.4x difference in actual risk!

Even at extreme α=1.0 (threshold 0.05 for Young Adult):
   DI only reaches 0.62 → Still not perfect fairness
   Because fairness = similar prediction rates, but reality differs

Options:
   ✓ α=0.00 (selected): Acc=87.83%, AGE DI=0.26 (hospital accepts age disparity)
   ✓ α=0.50: Acc=86.58%, AGE DI=0.48 (balanced trade-off, all 4 fair)
   ✓ α=1.00: Acc=82.82%, AGE DI=0.62 (aggressive fairness, significant accuracy loss)

Interpretation: This is HONEST about demographic realities. Perfect fairness when
true outcomes differ is mathematically impossible (violates performance constraints).

COMMON QUESTIONS & TROUBLESHOOTING
==================================================================================

Q1: Why does the notebook take 20-25 minutes?
A: GPU training of ensemble models:
   - LightGBM (1500 trees, GPU accelerated): ~4 minutes
   - XGBoost (1200 trees, GPU accelerated): ~3 minutes
   - Stacking (5-fold CV): ~2 minutes
   - AFCE framework (all phases): ~5 minutes
   → Total hardware-dependent, but typical on RTX 5070

Q2: Can I run this on CPU?
A: Yes, but slowdown factor ~10-20x
   - Change device='gpu' → device='cpu' in LGB/XGB configs
   - Overall runtime: 3-5 hours instead of 20-25 minutes
   - Not recommended unless GPU unavailable

Q3: What if I get CUDA out-of-memory errors?
A: Reduce batch sizes and tree counts:
   - LightGBM: n_estimators=800 (from 1500)
   - XGBoost: n_estimators=600 (from 1200)
   - DNN: batch_size=1024 (from 2048)
   - Note: May slightly reduce performance

Q4: Can I rerun just the AFCE section (Section 11)?
A: Yes! Skip cells 1-38, then run cells 39-47 (AFCE section)
   - Make sure you've already run the feature engineering & model cells
   - Cell dependencies are marked with comments

Q5: What's the difference between Standard and Detailed notebooks?
A: ONLY presentation:
   - Standard: Minimal markdown, code-focused
   - Detailed: Extensive markdown explanations, math, intuition
   - Code is identical, execution is identical, results are identical

Q6: How do I interpret the Pareto frontier?
A:  X-axis (alpha): AGE_GROUP fairness intensity
    - α=0.0: Minimal age correction (max accuracy 87.83%)
    - α=0.5: Balanced (middle ground, 86.58% accuracy)
    - α=1.0: Maximum age correction (low fairness for young adults, 82.82% accuracy)

    Choose based on hospital policy:
    - "Maximize accuracy": α=0.0 (our default)
    - "All groups equally fair": α=0.5 or α=0.7
    - "Extreme fairness": α=1.0

Q7: Why is RACE fairness improved more than SEX fairness?
A: Because RACE had more unfairness to correct:
   - RACE before: DI=0.618 (very unfair)
   - SEX before: DI=0.789 (borderline)
   - Threshold offsets can correct first one more
   - SEX was already close to fair

Q8: Can I use this model in production?
A: Not yet. Requires:
   1. Clinical validation (prospective study)
   2. FDA/CMS regulatory approval (medical device)
   3. Institutional Review Board (IRB) approval
   4. Fairness audit and monitoring systems
   5. Physician training and change management
   → This is research prototype stage, not clinical tool
